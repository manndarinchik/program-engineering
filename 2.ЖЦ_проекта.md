# 2. ЖЦ проекта

> Выбор и обоснование ЖЦ проекта. Нюансы ЖЦ.

## 2.1 Выбор жизненного цикла: Agile-модель

Для реализации проекта выбран гибкий (Agile) подход с использованием фреймворка Scrum.

## 2.2 Обоснование выбора Agile/Scrum

Scrum наиболее походит для быстрой итеративной разработки ПО с регулярными релизы. Учитывая целевую среду развертывания проекта, система мониторинга должна постоянно развиваться и дополняться, чтобы удовлетворять требованиям новых бизнес-прцессов.

## 2.4 Ключевые элементы процесса (Scrum)

- **Спринт**

  Итерация разработки с фиксированной длительностью.
  
- **Бэклог продукта**
  
  Упорядоченный по приоритету список всей функциональности, необходимой в продукте.

- **Планирование спринта**
  
  Событие по отбору задач из бэклога продукта в бэклог спринта.
  
- **Ежедневный стендап**
  
  Короткая встреча команды для синхронизации.
  
- **Обзор спринта**
  Демонстрация выполненной работы заинтересованным сторонам. 

- **Ретроспектива спринта**

  Встреча команды для анализа процесса и улучшений.
