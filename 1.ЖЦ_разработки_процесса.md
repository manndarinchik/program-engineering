# 1. Жизненный цикл разработки процесса

## 1.1. Выбор и обоснование жизненного цикла

Для разработки технического задания выбран жизненный цикл Agile с применением методики Kanban. Данная методика была выбрана ввиду отсутствия необходимости строго определенного версионирования и плана релиза. Кроме того, данный процесс наилучшим образом подходит для параллельного выполнения задач небольшой командой разработчиков. 

## 1.2. Принципы работы над документом

### 1.2.1. Организация рабочего процесса

Процесс организован вокруг трех ключевых эпиков (тематических блоков), каждый из которых содержит набор взаимосвязанных задач. Управление ведется с помощью виртуальных Kanban-досок со следующими стандартными колонками:

1. Open (Открыто): Задачи сформулированы, готовы к взятию в работу.
2. In progress (В работе): Задачи, над которыми ведется активная работа.
3. Review (На проверке): Завершенные задачи, ожидающие проверки и согласования.
4. Done (Выполнено): Принятые и завершенные задачи.

Разработка докумпентации разделена на 3 эпика:
- Структура управления проектом
    ```mermaid
    kanban
    Open
        [Формализация методик управления проектом]@{assigned: Чернавцев, priority: 'Normal'}
    [In progress]
        [Выбор жизненного цикла процесса разработки документации]@{assigned: Чернавцев, priority: 'High'}
        [Выбор жизненного цикла разработки проекта]@{assigned: Чернавцев, priority: 'High'}
    Review
    Done
    ```
- Управление артефактами проекта
    ```mermaid
    kanban
    Open
        [Формализация регламента версионирования проекта]@{assigned: Беликов}
        [Формализация регламента версионирования данных хранилищ информации]@{assigned: Беликов}
    [In progress]
        [Формализация списка артефактов проекта]@{assigned: Беликов}
    Review
    Done
    ```
- Управления процессом выпуска
    ```mermaid
    kanban
    Open
        [Формализация процессов тестирования]@{assigned: Брагин}
        [Формализация процедуры развертывания]@{assigned: Брагин}
    [In progress]
        [Формализация процедуры ввода в эксплуатацию]@{assigned: Брагин}
    Review
    Done
    ```
