# 1. ЖЦ разработки процесса

> __Выбор__ и __обоснование__ жизненного цикла, который будет использоваться для данного ТЗ (отдельный ЖЦ от самого проекта). Здесь описывается принцип работы над ним. Нужно определить роли участников и выстовить цели, которые этот документ должен выполнить. 

Основной процесс разработки документа - Kanban. Основное преимущество: паралелльная разработка нескольких частей документа. Свободный подход к внедрению, нету привязанности к спринтам.

Роли:
- Беликов Григорий - разработчик, менеджер потока
- Брагин Константин - разработчик
- Чернавцев Дмитрий - разработчик

Разработка докумпентации разделена на 3 эпика:
- Структура управления проектом
    ```mermaid
    kanban
    Open
        [Формализация методик управления проектом]@{assigned: Чернавцев, priority: 'Normal'}
    [In progress]
        [Выбор жизненного цикла процесса разработки документации]@{assigned: Чернавцев, priority: 'High'}
        [Выбор жизненного цикла разработки проекта]@{assigned: Чернавцев, priority: 'High'}
    Review
    Done
    ```
- Управление артефактами проекта
    ```mermaid
    kanban
    Open
        [Формализация регламента версионирования проекта]@{assigned: Беликов}
        [Формализация регламента версионирования данных хранилищ информации]@{assigned: Беликов}
    [In progress]
        [Формализация списка артефактов проекта]@{assigned: Беликов}
    Review
    Done
    ```
- Управления процессом выпуска
    ```mermaid
    kanban
    Open
        [Формализация процессов тестирования]@{assigned: Брагин}
        [Формализация процедуры развертывания]@{assigned: Брагин}
    [In progress]
        [Формализация процедуры ввода в эксплуатацию]@{assigned: Брагин}
    Review
    Done
    ```
